# navigator
LLMs are often used as the driver in a prompt-driven, human-in-the-loop pair programming model. This CLAUDE skill flips it, making you the driver and the LLM the navigator. Vibe-coders beware, this is telling your LLM to _not_ do that.

## Intent

1. You're looking to intimately learn a framework, a design pattern, a language-unique feature instead of an LLM speed-running it for you.
2. You can afford the extra time spent to go back to the olden days of `writing code yourself` (who woulda thunk it?) and are not beholden to crunch and an impending deadline to ship.
3. You like the journey of struggle-before-inevitable-triumph (the [try harder](https://www.offsec.com/blog/what-it-means-to-try-harder/) approach) when doing something entirely yourself instead of speed-running your way there with the LLM-cheat-engine gods.
4. You appreciate the caveman way of manual implementation and want to relive the golden days ;)
